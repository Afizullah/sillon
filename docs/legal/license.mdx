# La licence de votre produit

Que le code technique de votre produit soit en libre accès ou caché dans un coffre, il est important de choisir une licence adéquate pour que l'utilisation du code de votre produit reste dans un périmètre que vous acceptez. Nous vous conseillons de lire [ce document](https://gd4h-prod.s3.fr-par.scw.cloud/gd4h-prod/api_gd4h/files/17_Fiche_17_Les_licences_DESIGN.pdf) qui introduit l'usage des licences, et d'utiliser [le tableau de correspondances de data.gouv.fr](https://www.data.gouv.fr/en/pages/legal/licences/) pour choisir la licence la plus adaptée.

Nous aimerions juste apporter une clarification sur la licence `MIT` qui est très permissive :

- Cela fait sens pour une simple librairie technique ;
- Mais dans le cadre d'un produit complet, cela autorise une entreprise privée à tout reprendre, mettre son propre logo et le revendre à une entité publique qui n'aurait pas connaissance de l'outil public initial. _Vous pouvez limiter ce genre d'abus en adoptant la licence `AGPL-3.0`._

À noter que le contenu hors technique devrait être mis sous la licence de l'État `etalab-2.0`.
