# Décliner cette base de connaissances

Il est possible que vous soyez en accord avec une majorité de notre contenu au point de vouloir reprendre la base de connaissances et d'y apporter vos spécificités pour l'exposer en interne.

Nous n'avons pas de suggestions parfaites à vous proposer, mais nous sommes curieux d'avoir vos retours si vous en avez d'autres.

## Faire un fork complet du code

- \+
  - Flexibilité totale pour le modifier
- \-
  - Ne va pas bénéficier des corrections et améliorations facilement (il est toujours possible d'essayer de rebase depuis le code original mais cela peut se complexifier si on change la structure de présentation

## L'utiliser comme une documentation à tiroirs

- \+
  - En utilisant un outil de documentation comme Docusarus (ou autre) vous pourriez importer seulement les sections qui vous intéressent (actuellement dans des fichiers Markdown séparés) et écrire les vôtres à côté
- \-
  - Vous êtes cantonnés à des imports bruts des sections, sans granularité en leur sein
