# Le schéma de base de données

Nous privilégions l'utilisation d'un ORM pour représenter notre schéma de base de données, comme cela tout comme le code, le schéma est versionné.

Dans le cas où vous utilisez une base relationnelle (mentionné plus loin dans le chapitre ["base de données"](database-for-everything/relational-databases.mdx)), sachez qu'en utilisant un outil comme [DBeaver](https://dbeaver.io) vous êtes en mesure de générer un diagramme représentant votre schéma de données.

Pour cela via l'outil, connectez-vous à votre base de données, puis, dans l'arborescence clic droit sur votre base et _"Voir le diagramme"_.

<CaptionedImage src={DatabaseSchema} alt="" caption="Diagramme d'un schéma de base de données dans DBeaver" />

:::note
Comme celui-ci découle des relations déclarées par l'ORM dans votre code, il n'y a pas besoin de l'exporter pour le committer. Il reflète en permanence la réalité.
:::
