# Données sensibles

En tant que produit vous allez normalement héberger un minimum de données personnelles. Vous ne pouvez donc pas prendre un hébergeur lambda qui ne répond pas à un certain nombre de critères de confiance. Que ce soit [_légalement_](#la-protection-des-données) ou d'un point de vue [_sécurisation des données_](#sécurité-primaire).

Pour faire simple, si l'hébergeur affiche les standards suivants vous partez dans une bonne voie car c'est gage de qualité (compatible pour les données de santé…) :

1.  [_SecNumCloud_](https://www.ssi.gouv.fr/actualite/lanssi-actualise-le-referentiel-secnumcloud/) ;
2.  [_HDS_](https://fr.wikipedia.org/wiki/Certification_des_h%C3%A9bergeurs_de_donn%C3%A9es_de_sant%C3%A9) ;
3.  [_ISO 27001_](https://fr.wikipedia.org/wiki/ISO/CEI_27001) (grand minimum).

_À noter que [l'ANSSI référence les hébergeurs SecNumCloud](https://www.ssi.gouv.fr/entreprise/qualifications/prestataires-de-services-de-confiance-qualifies/prestataires-de-service-dinformatique-en-nuage-secnumcloud/), mais cela ne liste pas les petits hébergeurs qui sont basés sur de plus gros (ils ne gèrent pas l'accès physique aux bâtiments…)._

En avril 2023 [_l'État annonçait qu'il faut que tout hébergeur se conforme prochainement au SecNumCloud_](https://www.economie.gouv.fr/securite-performance-souverainete-strategie-cloud#D%C3%A9velopper%20un%20label%20%C2%AB%20cloud%20de%20confiance%20%C2%BB) pour continuer à héberger des services publics. Cela ne sera pas immédiat puisque certains hébergeurs ont un panel de produits assez large à adapter… Mais si votre hébergeur est HDS c'est déjà un très bon gage de sécurité.
