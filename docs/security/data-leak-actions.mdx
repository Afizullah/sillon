# En cas de fuite de données

Si vous constatez une fuite de données, le plus important est :

1.  Agir en limitant cette fuite et en changeant au plus vite les accès potentiellement corrompus ;
2.  Communiquer aux personnes les plus à même de vous aider sur le court terme :
    1.  À votre RSSI pour qu'il puisse vous accompagner ;
    2.  [À la CNIL s'il y a un risque pour la vie privée des personnes concernées](https://www.cnil.fr/fr/notifier-une-violation-de-donnees-personnelles) (et optionnellement aux personnes concernées par la fuite) ;
3.  Comprendre en détails comment cela a pu se produire et ce qui a été touché ;
4.  Documenter cette mauvaise expérience.

Quelques documents de l'ANSSI pour mieux se préparer et gérer une crise :

- [Anticiper et gérer sa communication](https://www.ssi.gouv.fr/uploads/2021/12/anssi-guide-communication_crise_cyber.pdf) _(2021)_ ;
- [Les clés d'une gestion opérationnelle et stratégique](https://www.ssi.gouv.fr/uploads/2021/12/anssi-guide-gestion_crise_cyber.pdf) _(2021)_ ;
- [Agilité & sécurité numériques](https://www.ssi.gouv.fr/uploads/2018/11/guide-securite-numerique-agile-anssi-pa-v1.pdf) _(2018)_.

:::caution
Si vous avez laissé en accès public des données privées (brièvement ou non), il faut réagir comme si c'était une fuite de données. Présupposer que personne n'ait accédé aux données exposées est une erreur.
:::
